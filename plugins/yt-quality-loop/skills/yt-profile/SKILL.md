# YT Channel Profile

「運営者の思い通りの台本」をループで出すための下ごしらえです。ループは**言語化されたものにしか収束しません**。このスキルは、頭の中にある「うちのチャンネルらしさ」を、作る係が従える指示と、採点係が測れるものさしに翻訳して `.yt-loop/channel-profile.md` に保存します。

## Step 0: モード判定

```bash
ls .yt-loop/channel-profile.md 2>/dev/null && echo "MODE:update" || echo "MODE:create"
```

- **create**: Step 1 から順に実行
- **update**: 既存プロファイルを Read してから、差分ヒアリングのみ行う (「前回の納品物で直した箇所はありますか?」「新しく伸びた動画はありますか?」)。直しの内容は **セクション 8 (直しの履歴)** に「次回への指示」の形で追記し、恒常的なルールに昇格すべきものはセクション 3/4/6 に反映する

## Step 1: 材料集め

ユーザーに確認する (1 回のやりとりにまとめる):

1. **過去台本 or 文字起こしの場所** — `$ARGUMENTS` にパスがあればそれを使う。3-10 本、**伸びた動画のもの優先**。「作業フォルダに『過去台本』フォルダを作ってテキストを入れる → そのフォルダをこのチャットにドラッグ」という動線で案内する (パスという概念を前提にしない)
2. 台本を用意できない/面倒な場合は **最小プロファイル**で始める: 質問は 2 つだけ (「誰向けのチャンネルで、視聴者は何に困っていますか?」「絶対に言わないこと・避けたい表現は?」)。残りのセクションは「未設定」のまま生成し、「納品物を手直しするたびに `/yt-profile 更新` で育てるのが一番速い」と伝える
3. 既に**自作の台本スキル**を持っている場合、その SKILL.md 等を材料として渡してもらってよい — スキルに書かれたこだわり・基準・口調の記述をプロファイルへ移植する (移植元は作成ソースに記録)
4. YouTube 分析系の MCP ツール (vidiq 等) がこのセッションで使える場合のみ、上位動画の特定や文字起こし取得に使ってよい (使えない環境を前提に、無くても完走すること)

## Step 2: 過去台本の分析 (ファイルがある場合)

各台本を Read し、**複数本に共通するパターンだけ**を抽出する (1 本だけの特徴は採用しない。ただし「らしさ」が際立つフレーズはお手本フレーズ集の候補にする):

- **口調・文体**: 一人称、視聴者の呼び方、語尾、口癖・型フレーズ、1 文の長さ (count-chars.sh と文数で実測)、漢字/ひらがなの傾向
- **構成の型**: 冒頭何文目までに何が来るか (フックの型)、セクションの数と遷移、まとめと CTA の位置・形
- **中身の傾向**: 具体例の出し方 (体験談? 画面実演? 数字?)、主張の深さ、扱うテーマの範囲

## Step 3: ヒアリング (足りない項目だけ、最大 5 問)

分析で埋まらなかった項目のみ聞く。候補:

1. ターゲット視聴者は誰で、何に困っていて、見終わって何ができるようになってほしい?
2. 全動画に共通する「チャンネルの約束」は? (何をしないチャンネルか、も)
3. 絶対に言わないこと・避けたい表現は?
4. 伸びた動画とその理由の仮説は? (維持率・CTR などデータがあれば)
5. 台本の使い方は? (そのまま読む / 目次として使う / テロップ前提)

## Step 4: プロファイル生成

同梱の `profile-template.md` を読み、全項目を埋めて `.yt-loop/channel-profile.md` に Write する:

```bash
mkdir -p .yt-loop
cat "${CLAUDE_PLUGIN_ROOT}/skills/yt-profile/profile-template.md"
```

**書き方の規律:**

- 全項目を「台本を書く人への実行可能な指示」として書く。形容詞 (「親しみやすく」) は禁止 — 「語尾は です/ます。断定は『〜なんです』」のように書く
- **お手本フレーズ集は過去台本からの原文引用のみ。** 創作しない
- **分からない項目は「未設定」と書く。** 埋めるための創作は、存在しない「らしさ」に採点係が引きずられる事故になる
- 勝ちパターンメモは、データの根拠がある行とただの仮説の行を区別して書く

## Step 4.5: 機械チェックルールの書き出し (任意)

プロファイルの「禁止の言い回し」「NG リスト」のうち、**単語の出現として○×判定できるものだけ**を `.yt-loop/mechanical-checks.json` に書き出す:

```json
{
  "min_chars": 0,
  "max_chars": 0,
  "count_mode": "spoken",
  "forbidden_words": ["いかがでしたか", "絶対に儲かる"],
  "max_ending_streak": 2
}
```

`count_mode: "spoken"` にすると、演出メモ (【】)・見出し・コメント・空白を除いた「読み上げに乗る文字数」で判定する (演出メモでの文字数水増し対策。python3 が無い環境では raw に縮退)。

- `min_chars` / `max_chars`: 毎回同じ尺で作るチャンネルのみ設定 (それ以外は 0 = 無効)
- `forbidden_words`: 出現したら即差し戻しの単語のみ。文脈依存の表現 (トーン等) は入れない — それは採点係の仕事
- これがあると `/yt-loop` は採点係に渡す前にスクリプトで○×を検査し、NG なら採点なしで差し戻す (トークン節約 + AI が言い張っても覆らない)

## Step 5: 確認と確定

生成したプロファイルをユーザーに見せ、修正を 1 回受けて確定する。最後に効き方を説明する:

- 次回の `/yt-loop` から、台本系タスクは自動で **チャンネル適合版の採点係 (assign-yt-channel-evaluator)** に切り替わり、「構成の型の遵守」「口調・文体の一致」が採点軸に入る
- 作る係にはお手本フレーズと NG リストが渡る
- **納品物を手直しした時は `/yt-profile 更新` で還流させる** — 直しがセクション 8 に貯まり、次のループから反映される。これを 2-3 回回すと「思い通り」に急速に近づく

## ルール

- 出典の分からない一般的な「良い台本論」でプロファイルを埋めない。**このチャンネルの実物と本人の言葉だけ**から作る (一般論はプリセット採点係が既に持っている — ここに書くのは差分だけ)
- プロファイルは `.yt-loop/channel-profile.md` の 1 ファイルのみ。分散させない

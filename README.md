# yt-quality-loop

**YouTube 向けの成果物 (台本 / ショート台本 / タイトル・サムネ案 / 企画 / 概要欄) を、品質スコアが合格点に達するまで AI が自動で磨き込むプラグイン**です。Claude Code・Codex・Cursor・Antigravity で使えます。

「作って」と 1 回頼んで終わりではなく、**作る係 → 採点する係 → 直す** のループを、合格点 (デフォルト 90 点) に届くまで自動で回します。続行するかどうかの判定は AI の気分ではなく、**シェルスクリプトの整数比較** が行います。だから疲れず、迎合せず、毎回同じ基準で止まります。

```
あなた:「AI初心者向けに『ChatGPTの始め方』10分動画の台本を作って」
        │
        ▼
  ①計画 → ②生成 → ③機械チェック → ④採点 (まっさらな別の頭が実物だけを見て採点)
        ▲                                │
        │        89点 < 90点             ▼
        └── 直し方付きで差し戻し ──── 90点以上 → 検証して納品 ✅
```

## 4 つの停止条件

| 条件 | デフォルト | 役割 |
|---|---|---|
| 品質スコア | 90 点以上で合格 (検証付き) | 目標達成で止まる |
| 最大回数 | 6 回 | 伸びない時に諦める |
| 最大時間 | 120 分 | 暴走の天井 |
| 進捗ゼロ | 2 回連続でスコアが上がらない | 堂々巡りにお金を使わない |

どう終わっても、**それまでのベスト版は必ず納品されます**。

## 時間とお金の目安 (実測ベース)

| タスク | 1 周の実測目安 |
|---|---|
| 概要欄・タイトル案など短いもの | 約 10〜15 分 |
| 台本 + プロファイル (6 軸採点 + 合格時の確認採点) | 約 15〜25 分 |

- **多くの場合 2〜4 周で終わります。** コストの上限は「1 周ぶん × max 回数」。不安なら `max: 3` から
- threshold の実測感: チャンネル適合 6 軸で **94 はかなり厳しめ** (実測ではベスト 92 で max 終了)。**初回は 90 がおすすめ**
- **90 点は「台本の質の点」であって、再生数の保証ではありません。** プリセットの採点軸は解説・ハウツー系チャンネル向けに作ってあります (合わない人は下の「プリセットが合わない人へ」)

## インストール (Claude Code)

**0. このフォルダを入手する** — 配布 zip を解凍、または `git clone`。以降の `<フォルダのパス>` は、**Finder からフォルダをターミナルにドラッグ&ドロップ**すると自動で入力されます。

**1. jq を入れる** (JSON 処理ツール・必須):

```bash
brew install jq
```

Homebrew が未導入の Mac は先に [brew.sh](https://brew.sh) の 1 行インストールを実行してください。Windows は WSL (Ubuntu) 上で `sudo apt install jq` — WSL を使えない場合、現状このプラグインは不向きです。

**2. プラグインを入れる**:

```bash
claude plugin marketplace add <フォルダのパス>
cd ~/YouTube作業フォルダ
claude plugin install yt-quality-loop@yt-quality-loop --scope project
```

**3. Claude Code を再起動して `/yt-doctor`** — 全部 ○ なら準備完了です。

**4. 初回だけ**: ループ中に「Bash を実行していいですか?」という許可を何度か聞かれます。**「常に許可」を選ぶと止まらずに回ります** (聞かれるのは jq などのループ制御コマンドです)。

## 使い方

```
/yt-loop AI初心者向けに「ChatGPTの始め方」10分動画の台本。結論先行、専門語は言い換え付き
```

オプション付き:

```
/yt-loop 「朝のルーティン」ショート台本 60秒 (threshold: 92, max: 8)
```

開始時に「最大何周・目安何分・止め方」が表示され、あとは放置。合格するか上限に達すると、ベスト版が `yt-loop-output-*.md` に納品されます。

### コマンド一覧

| コマンド | 用途 |
|---|---|
| `/yt-loop <作りたいもの>` | 品質ループを開始。合格するか上限に達するまで自動で回る |
| `/yt-profile` | チャンネルプロファイル (台本の「らしさ」のものさし) を作成・更新する |
| `/yt-import-skill <SKILL.md>` | 既存の台本スキル/プロンプトを作る係とプロファイルへ移植する |
| `/yt-doctor` | セットアップ診断。「動かない」時はまずこれ |
| `/yt-loop-cancel` | 実行中のループを止める (それまでのベスト成果物は残る) |

設計方針: **コマンドはこれ以上増やしません。** 新しい機能 (動画ブリーフ、直しからの学習など) は `/yt-loop` と `/yt-profile` の中のモードとして入ります — 覚えることを増やさないため。

### 採点係のプリセット (自動選択)

| 作りたいもの | 採点軸 (固定) |
|---|---|
| 長尺動画の台本 | 冒頭フック / 視聴維持設計 / 構成の明確さ / 具体性と信頼性 / CTAと導線 |
| 長尺台本 + プロファイルあり | 上記 + **構成の型の遵守 / 口調・文体の一致** (チャンネル適合 6 軸) |
| ショート台本 | 冒頭2秒フック / テンポ密度 / オチとループ性 / 尺と文字数規律 |
| タイトル・サムネ案 | クリック誘引力 / 内容整合(釣り耐性) / 検索・推薦適合 / 文字数と可読性 |
| 企画・ネタ出し | ターゲット適合 / 差別化 / 需要の根拠 / タイトル・サムネの立てやすさ |
| それ以外 (概要欄など) | タスクに応じて 3-5 軸を自動設定 |

採点軸は**ループの途中で変わりません**。「基準を緩めて合格する」「採点を自書きする」という AI のズル (グッドハートの法則) は、スキーマ検証・合格時の再検証・ものさしの指紋照合が機械的に拒否します。ループ開始時に「どの採点係のどの軸で採点するか」が必ず表示されます — 知らないうちに他人のものさしで採点されることはありません。

### プリセットが合わない人へ (エンタメ・Vlog・考察系など)

プリセットの軸と重みは、解説・ハウツー系の世界観で作った「最初の床」にすぎません。合わなければ**全部あなたの軸に入れ替えられます**:

```
/yt-loop 台本: 深夜ラジオ風トーク回 (evaluator: assign-yt-evaluator, criteria: 世界観の没入, 余白とテンポ, 語りの温度, 尺の規律)
```

自由軸で回すと、ループ開始時に**軸ごとの採点アンカー (90 点/75 点の目盛り) が起草・固定**され、以降の採点はその目盛りで安定します。アンカーも改ざん検知の対象で、最終報告にパスが出るので、次回は `anchors: <パス>` で再利用できます。

なお、このツールに入っている「意見」は 2 種類あり、扱いが違います:

- **数値の経験則** (タイトル 28-32 字、60 秒 ≒ 250-350 字など) — ただの初期値です。`criteria:` で自由に置き換えてください
- **視聴者ファーストの下限** (釣りタイトルは総合 60 上限、煽って終わる台本の減点など) — これは**このツールの思想**です。外したい場合は採点係の自作が必要で、意図的に摩擦を残しています

### 「思い通りの台本」に近づける — チャンネルプロファイル

そのまま回すと「一般的に良い台本」に収束します。「うちのチャンネルらしい台本」に収束させるには、らしさを**ものさし化**します。ループは言語化されたものにしか収束しません。

```
① /yt-profile                      ← 過去台本を渡す (無ければ質問2つの最小プロファイルでOK)
      ↓ .yt-loop/channel-profile.md が生まれる
      (ターゲット / 構成の型 / 口調・口癖 / お手本フレーズ原文 / NGリスト / 勝ちパターン / 納品フォーマット)

② /yt-loop 台本: ...               ← プロファイルを検出すると採点係が自動で
      ↓                              「チャンネル適合版 (6軸)」に切り替わる

③ 納品物を手直ししたら /yt-profile 更新   ← 直しが「次回への指示」として貯まる
```

③は**手直しした後のファイルを渡すだけ**でも動きます — AI が納品物との差分を取り、「次回への指示」に翻訳して確認を求めます (同じ直しが 2 回現れた時だけ恒常ルールに昇格。1 回きりの気まぐれは固定しません)。

③の還流を 2-3 回まわすと、手直しの量が目に見えて減っていきます。**プロファイルの質 = 「思い通り」度の上限**です。特に効くのは「お手本フレーズ集 (原文引用)」と「NG リスト」— 形容詞 (「親しみやすく」) は効きません。記入の粒度は [記入例](docs/examples/channel-profile-example.md) が手本になります (実走 E2E でチャンネル適合 6 軸採点を実際に駆動した実績のあるプロファイルです)。

### 動画ブリーフ — 1 本ごとの設計書

プロファイルが「チャンネルの憲法」なら、ブリーフは「この動画 1 本の約束」です。台本系タスクではループ開始前に自動で作られます (`.yt-loop/briefs/`):

- **誰向けか / 約束する変化 (Before→After) / 絶対入れる話 / 絶対言わない話 / 視聴後にしてほしい行動 1 つ**
- 採点係は「約束した変化を台本が回収したか」を採点根拠にします。**仮タイトルが決まっているならブリーフに書いてください** — タイトルが煽った疑問に台本が答えない「釣り」を採点で弾けます
- ブリーフも指紋照合の対象です (ループ中に「言わない話」を消して合格する抜け道はありません)

### 1 本の動画をまるごと作るレシピ

台本 → タイトル → 概要欄は、納品物を次の task で指名して連続で回します (自動連結モードは意図的に作っていません — 1 回の失敗が 3 倍に膨らむため):

```
/yt-loop 台本: ○○ (10分)
→ 納品された yt-loop-output-XXXX.md を確認
/yt-loop タイトル案10本とサムネ文言: 台本 yt-loop-output-XXXX.md の内容で
/yt-loop 概要欄: 同じ台本で。検索キーワードは「○○」
```

タイトルを先に決める流儀なら順序を逆に — 決めたタイトルをブリーフ (仮タイトル欄) に入れて台本ループを回すと、採点係が「タイトルの約束を台本が回収したか」まで見ます。

**冒頭 15 秒だけ磨くレシピ** (専用コマンドはありません — task の切り方で足ります):

```
/yt-loop 台本 yt-loop-output-XXXX.md の冒頭15秒 (約80字) だけを磨く。本編は変えない。
(criteria: 冒頭2秒の一文目, 損失/欲望の名指し, タイトルの約束との接続, 80字規律, threshold: 92, max: 4)
```

### 既に自分の台本スキルを持っている人へ

そのスキルは捨てません。**「作る係」の席にそのまま差し込めます**:

```
/yt-loop 台本: 新NISAの解説 10分 (generator: my-script-skill)
```

あなたのスキルが書き、ループの採点係が検品し、不合格なら直し方付きで差し戻されます。スキル＝専門知識の移植、ループ＝合格までの収束。スキルはループの部品として生き続けます。スキルに書き込んだこだわり・基準の記述は、`/yt-profile` に渡せばプロファイルへ移植もできます。

既存スキルを整理してから使う場合:

```
/yt-import-skill path/to/my-script-skill/SKILL.md
```

移植メモは `.yt-loop/imported-generators/` に保存され、`generator:` に指定する名前、プロファイルへ移すべきルール、除外すべき危険指示が分かれます。詳しくは [docs/existing-skill-integration.md](docs/existing-skill-integration.md)。

## デモ

運営者に見せる時は、先に [docs/demo-youtube-script-loop.md](docs/demo-youtube-script-loop.md) を使ってください。1周目の弱い台本、採点 feedback、2周目の改善、最終レポートの見え方まで 1 ページで説明できます。

## Codex / Cursor / Antigravity で使う

Codex / Cursor はプラグイン形式で、Antigravity は `plugin.json` パッケージとして同梱しています (詳細: [codex/README.md](codex/README.md)):

```bash
# Codex (プラグイン)
codex plugin marketplace add <このフォルダのパス or GitHubの owner/repo>

# Cursor: マーケットプレイス形式 (.cursor-plugin/ + cursor-plugin/) を同梱
# Antigravity: plugin.json 形式の antigravity-plugin/ を同梱 (実験的)

# プラグインが使えない環境向け (スキルを直接コピー):
bash codex/install-skills.sh all
```

Codex には `$yt-loop-hook` (Stop hook 駆動) と `$yt-loop` (hook なしフォールバック) の 2 経路を同梱しています。Codex の plugin hooks はインストールだけでは自動信頼されないため、hook 定義を確認・信頼してから `$yt-loop-hook` を使ってください。信頼しない場合や Cursor / Antigravity では `$yt-loop` を使います。

3 環境とも採点係のサブエージェント / custom agent 定義を同梱しています。自己採点は通常経路ではなく、fresh な採点係が使えなかった場合だけの最終フォールバックで、最終報告に開示されます。

無人実行 (ターミナルから直接、codex CLI 必須):

```bash
bash codex/yt-loop-runner.sh "<task>" [threshold] [max] [criteria]
```

## よくある質問

**Q. 動かない / ループが続かない**
まず `/yt-doctor`。jq の有無、hook の動作、フォルダの権限を○×で確認できます。インストール直後なら Claude Code の再起動を忘れずに。

**Q. ループが止まらなくなった / 様子がおかしい**
`/yt-loop-cancel` で止められます。それまでのベスト成果物のパスが表示されます。

**Q. 点数が伸びずに終わった**
ベスト版は納品されます。多くの場合、直すべきはループの回数ではなく **task の書き方**です。「誰向け / 長さ / 何を入れるか / 何ができたら完成か」を task に足して再実行してください。2 回連続でスコアが上がらないと自動停止するのは、堂々巡りにお金を使わないためです。

**Q. 毎回 90 点が必要?**
いいえ。`threshold: 80` のように下げられます。数をこなす下書き用途なら 80、公開物の最終稿なら 90-92 が目安です。

**Q. 文字数や禁止ワードを毎回確実にチェックしたい**
`/yt-profile` を作ると、○×で判定できる基準が `.yt-loop/mechanical-checks.json` に書き出され、採点の前にスクリプトが機械判定します (NG なら採点なしで差し戻し)。○×で決まることは AI ではなく機械に任せるのが安くて確実です。

**Q. (上級者向け) 採点係を自作したい**
`skills/assign-yt-channel-evaluator/` をコピーして SKILL.md と eval-schema.json を書き換えれば専用の採点係が作れます。`mechanical-checks.json` の手書きも可能です。ただし通常は `/yt-profile` 経由で足ります — JSON を手で書くと壊れやすいので、まずプロファイルから。

## フォルダ構成

```
yt-quality-loop/
├── README.md / CHANGES.md
├── sync-packages.sh                    ← スキル正本を各プラグインへ同期
├── scripts/                            ← 配布前検証 (validate-packages / e2e-smoke)
├── docs/                               ← デモ、E2E手順、既存スキル連携ガイド
├── .claude-plugin/marketplace.json     ← Claude Code マーケットプレイス
├── plugins/yt-quality-loop/            ← Claude Code プラグイン本体
│   ├── hooks/hooks.json                ← UserPromptSubmit / Stop
│   ├── scripts/                        ← ループ制御 (開始/判定/検証/機械チェック/指紋/診断)
│   └── skills/                         ← 12 スキル (起動/補助5 + 作る係1 + 採点係6)
├── .agents/plugins/marketplace.json    ← Codex マーケットプレイス
├── codex-plugin/                       ← Codex プラグイン (skills + hooks + scripts + agent 定義)
├── .cursor-plugin/marketplace.json     ← Cursor マーケットプレイス
├── cursor-plugin/                      ← Cursor プラグイン (skills + agents)
├── antigravity-plugin/                 ← Antigravity プラグイン (plugin.json + skills + agents。実験的)
├── agents/                             ← Cursor / Antigravity 共通 evaluator agent 正本
└── codex/                              ← スキル正本 + Codex agent/hook 正本 + インストーラ + 無人ランナー
```

## 検証

```bash
bash scripts/validate-packages.sh
bash scripts/e2e-smoke.sh
claude plugin validate plugins/yt-quality-loop --strict
bash sync-packages.sh
```

配布前の確認手順は [docs/e2e-checklist.md](docs/e2e-checklist.md) にまとめています。

配布物に他人のスキル/プラグインを混ぜる時は、中身を一度読んでから。あなたのループに入るものは、あなたが読む。

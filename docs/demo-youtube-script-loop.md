# デモ: 台本が品質ループでどう良くなるか

このデモは、YouTube 運営者に `yt-quality-loop` の価値を説明するためのサンプルです。実際のチャンネル情報や秘匿ノウハウは含めていません。

## 依頼例

```text
/yt-loop AI初心者向けに「ChatGPTの始め方」10分動画の台本。
視聴者は40代の個人事業主。専門語は言い換え、最後に次の動画への導線を入れる。
threshold: 90, max: 4
```

## 採点軸

長尺台本として自動選択:

- 冒頭フック
- 視聴維持設計
- 構成の明確さ
- 具体性と信頼性
- CTAと導線

チャンネルプロファイルがある場合は、台本系タスクでは「構成の型の遵守」「口調・文体の一致」も評価に入ります。

## 1周目のよくある状態

スコア例: `82`

良い点:

- ChatGPT の始め方というテーマは分かる
- 基本手順は並んでいる
- 初心者向けの言い換えはある

不足しやすい点:

- 冒頭が説明から入り、視聴者の悩みを刺していない
- 「登録して試す」だけで、最初の1プロンプトまで導けていない
- 途中離脱を防ぐ小さな予告や回収が弱い
- CTA が一般的で、次動画を見る理由が薄い

採点係の feedback 例:

```text
冒頭フックは「ChatGPTを始めたいが何を聞けばいいか分からない」という悩みを一文目で名指ししてください。
手順は登録方法だけでなく、最初に入力する具体プロンプトまで落とす必要があります。
各セクション末に「次で何が解決するか」を入れ、最後のCTAは次動画の内容とつなげてください。
```

## 2周目の改善例

改善内容:

- 1文目を視聴者の悩みに変更
- 登録後に最初に入れるプロンプトを3例追加
- 各セクション末に次への引きを追加
- CTA を「次の動画を見る理由」に変更

スコア例: `91`

```text
「ChatGPTを開いたのに、最初の一言が出てこない」
もしここで止まっているなら、問題はあなたの理解力ではありません。
最初に聞く型を知らないだけです。
この動画では、登録より先に大事な「最初の質問の作り方」まで、今日そのまま使える形で説明します。
```

## 運営者への見せ方

説明の軸は「AIに台本を書かせる」ではなく、「合格基準を渡して、作成・自己チェック・修正を回す」です。

特に伝えるべきこと:

- 最初の出力を信じない。必ず採点係を分ける
- 伸びる保証ではなく、公開前の品質ムラを減らす仕組み
- 思い通りにしたいなら、チャンネルプロファイルを作る
- 既存の台本スキルは捨てず、作る係として差し込む

## v1.5 以降の新要素 (デモで触れると刺さる 3 点)

1. **開始宣言** — ループが回り始める前に「採点係と軸・ブリーフの約束・アンカーのパス」が必ず表示される。「何のものさしで採点されるかが常に見える」ことを最初に見せると安心感が違う
2. **動画ブリーフ** — 台本タスクでは 1 本ごとの設計書 (誰向け / 約束する変化 / 絶対言わない話 / 視聴後の行動) が自動生成され、採点係が「約束の回収」まで見る。仮タイトルを入れると「タイトルの約束を台本が回収したか」も採点される
3. **合格時の確認採点** — 合格点が出た時だけ、まっさらな採点係がもう 1 回採点して**低い方を採用**する (まぐれ合格の防止)。機械がこの実在まで検証していることを一言添えると、「甘い自己採点で通る」不安への答えになる
